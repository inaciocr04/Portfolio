SAE 105

Je me suis conformé aux standards du Web et aux normes d'accessibilité :

Expliquez pourquoi :

Comment pouvez-vous le prouver ?

Pour respecter les standards du Web et les normes d'accessibilité, j'ai dû utiliser le validateur W3C pour savoir si le HTML et le CSS étaient corrects afin que le site soit publié. J'ai respecté la norme Web sur les caractères spéciaux comme des Majuscules ou des accents ce qui gêne à la lecture des fichiers pour l'hébergeur du site. J'ai respecté la norme UTF8 afin que mon langage soit bien encodé pour le Web. J'ai modifié les dimensions des images afin d'optimiser le poids de mon site inférieur à 200ko max par images et leurs types donc (png, jpg, webp, svg), avec des chemins relatifs pour lire les images. Tout mon dossier pour se site fait 13,8Mo.

Ensuite toujours dans les standards du Web, il fallait mettre de la sémantique donc les balises :

  • header /header
  • nav /nav
  • main /main
  • aside /aside
  • footer /footer
  • article /article
  • section /section

Toutes ces balises sont apparentes mais pour ma part je n'ai pas mis de balise « section » mais j'ai plutôt mis des balises « div »

J'ai produit du code fonctionnel, sobre et réutilisable :

Expliquez pourquoi :

  • réutilisable : mon code est-il lisible par un autre développeur ?
  • fonctionnel : mon code et les fichiers ressources sont-ils correctement rangés ?
  • sobre :comment ai-je réussi à éviter de dupliquer du code (copier/coller) ?

Mon code est plutôt lisible par chaque développeur qui viendrait lire mon code. Il le comprendrait normalement, pour cela j'ai mis des commentaires dans le HTML et surtout dans le CSS afin que l'on puisse comprendre à quoi correspond chaque ligne de code. Justement pour éviter les répétitions, j'ai utilisé un générateur de templates qui consiste à créer un fichier .TWIG qui va être le squelette d'un article et qui va pouvoir être répéter avec des boucles 'for' et le contenu des boucles for est lu dans un fichier PHP qui est notre data. C'est un endroit où tous mes textes et images s'y retrouvent. Pour être ordonner, il fallait bien classer tous mes fichiers alors j'ai mis tous les CSS ensembles dans un dossier et j'ai un CSS par pages. Ensuite j'ai mis toutes mes templates réunies dans un dossier 'templates' unique (.twig) qui correspond à chaque squelette de pages. On a un dossier avec toutes les images utilisées, nous avons tous mes fichiers data.php qui sont dans un dossier 'include' où il y a mes fichiers data de chaque pages que se soit en portugais ou en français et pour finir tout cela est rangé dans un dossier qui réunit l'ensemble de mes dossiers. Tous mes PHP sont directement dans ce fichier et pas un autre pour que le navigateur les lise correctement.

Je suis autonome sur mon environnement de développement :

Quelques outils, logiciels, plugin avez-vous utilisés ?

Pour effectuer tout cela, j'ai employé l'éditeur de texte « Visual Studio code » qui nous permet d'écrire du code en HTML, CSS et PHP. Ensuite j'ai quelques extensions de Visual studio code qui sont utiles comme : ' Remote SSH ' : il permet d'utiliser n'importe quelle machine distante avec un serveur SSH comme environnement de développement. ' Live server ' : il permet de voir l'avancement de sa page web en local sur la machine ' Google translate ' : il permet de traduire notre texte dans la langue demandée.

J'ai produit des pages Web fluides incluant un balisage sémantique efficace :

Mon code HTML est-il sémantiquement efficace ?

Mes styles CSS sont-ils efficaces ?

Personnellement ma sémantique est bien structurée comme demandé car on distingue bien la différence entre chaque partie de mon code. En ce qui concerne mon CSS j'en ai un par page qui est structuré par des commentaires afin de bien comprendre mon code et chaque code est utile pour les pages.

J'ai généré des pages Web à partir de données structurées :

Quels types de données ai-je utilisé ?

Quels algorithmes ai-je développé ?

En type de données utilisées, j'ai utilisé des blocks, des tableaux, beaucoup de for pour que tout soit modifiable. Ensuite l'algorithme que j'ai développé à la possibilité de tout modifier directement dans mes data base afin que rien ne bouge sauf le contenu : image, texte, tableaux. Pour cela j'ai un fichier TWIG principal qui va donner pour chaque page un header, un nav et un footer similaire qui se répètera sur chaque page grâce à : {% extends "base.twig" %} Et ensuite dans le base.twig, il y a un {% block main %} / {% endblock main %} qui va être dans chaque fichier où le contenu changera et pour cela on a les fichiers data qui comporteront les données texte et images pour le site.

J'ai mis en ligne mon site Web en utilisant une solution d'hébergement standard :

Quels outils ai-je utilisé ?

Une preuve peut être une notice d'installation en quelques phrases.

J'ai utilisé l'hébergeur Web qui est le serveur TP2 de l'université donc qui est un serveur en ligne où seuls, les étudiants et les professeurs peuvent lire le site. Pour cela, j'ai utilisé « XAMP » qui est un serveur Web en local donc un serveur hébergé par ma machine où seulement moi et les personnes ayant l'accès peuvent le lire si j'ouvre le serveur. Pour cette SAE j'ai exploité principalement XAMP donc une fois installé, j'ai mis tous mes fichiers structurés dans le dossier « htdocs » qui va être lu par XAMP si je lance le serveur en local. Et pour le mettre en ligne sur TP2, j'ai déplacé mes fichiers dans un dossier sur mon site TP2 dans un dossier SAE105.
